🏠 1. Présentation de Sallon-ConnecT
Comprendre ce qu'est Sallon-ConnecT, ce qu'il peut faire, et où il tourne.

Sallon-ConnecT est un tableau de bord local pour suivre et piloter les fonctions du salon connecté depuis un navigateur. Il centralise les appareils, les médias, les scénarios, les notifications, les tâches, les profils et les sauvegardes.

La plateforme fonctionne sur votre machine et votre réseau local. Elle ne nécessite pas de cloud.

Ce que vous pouvez faire

  • Ouvrir un dashboard depuis un PC, une tablette ou une TV
  • Naviguer entre Hub, Appareils, Agents, Médias, Scénarios, Notifications, Observabilité, Tâches, Profils et Sauvegarde
  • Installer l'interface comme application PWA
  • Utiliser le Mode TV pour un affichage plein écran
  • Créer des sauvegardes locales contrôlées
  • Consulter les notifications locales et les résultats de tâches

Adresses locales

ServiceAdresse
Backend Expresshttp://localhost:3000
Frontend Next.jshttp://localhost:3001
📋 2. Prérequis
Vérifier que votre machine est prête avant d'installer.
  • Windows 10 ou 11

  • npm (installé avec Node.js)
  • PowerShell disponible
  • Ports 3000 et 3001 libres

powershell

npm --version
✓ Résultat attendu : v22.x.x et 10.x.x ou supérieur
⚙ 3. Installation
Installer Sallon-ConnecT sur Windows avec les scripts fournis.

3.1 Installation guidée (recommandée)

powershell
scripts\windows\install\install-sallon-connect.bat

Le script effectue automatiquement :

  • Vérification des prérequis
  • Création de runtime/, logs/, backups/ et dist/
  • Création de .env depuis .env.example si absent
  • Installation des dépendances (npm install)
  • Build du frontend (npm run build:frontend)
  • Création des raccourcis Bureau et Menu Démarrer

Options utiles

powershell
powershell -ExecutionPolicy Bypass -File scripts\windows\install\install-sallon-connect.ps1 -SkipInstallDeps
powershell -ExecutionPolicy Bypass -File scripts\windows\install\install-sallon-connect.ps1 -SkipBuild
powershell -ExecutionPolicy Bypass -File scripts\windows\install\install-sallon-connect.ps1 -NoShortcut

3.2 Vérifier les prérequis seuls

powershell
powershell -ExecutionPolicy Bypass -File scripts\windows\install\check-prerequisites.ps1

3.3 Premier lancement (wizard)

powershell
scripts\windows\install\first-run-wizard.bat
⚠ Si vous activez SmartThings, modifiez .env manuellement — ne partagez jamais son contenu.

📺 6. Mode TV
Afficher le dashboard en plein écran sur une TV ou un grand moniteur.

Le Mode TV adapte le dashboard à un affichage plein écran. Cliquez sur Mode TV dans la barre du haut, ou utilisez la touche T.

Raccourcis clavier

TActiver / désactiver le Mode TV FPlein écran / quitter le plein écran RActualiser NAller aux notifications SAller aux tâches HAller à l'observabilité ÉchapQuitter un panneau actif

Comportement de sécurité

🔒 Aucun raccourci ne lance une commande sensible — aucune scène SmartThings, aucune commande TV, aucun streaming automatique.

Conseils

  • Démarrez d'abord le backend et le frontend
  • Ouvrez http://localhost:3001 sur l'écran cible
  • Utilisez le profil TV pour une interface plus restrictive
🎤 7. Assistant Vocal
Piloter la navigation du dashboard avec des commandes vocales locales.

Activer l'assistant

  1. Démarrez le backend et le frontend
  2. Ouvrez http://localhost:3001
  3. Cliquez sur Assistant vocal dans la barre du haut
  4. Cliquez sur Micro pour démarrer l'écoute
L'écoute ne démarre jamais automatiquement — elle doit être déclenchée par un clic utilisateur.

Commandes disponibles

CommandeAction
ouvre le hub / les appareils / les agents…Navigation section
actualiseRafraîchir le dashboard
scanne le statutScan des appareils
active / désactive le mode TVMode TV
plein écran / quitte le plein écranPlein écran
statut du systèmeAfficher le statut

Commandes refusées (bloquées)

🔒 Bloquées : allumer/éteindre la TV · scènes SmartThings · streaming · restauration sauvegarde · suppression audit · vidage notifications · changement de profil

Sécurité

  • Aucun cloud — pas d'OpenAI, Google Cloud, Azure ou Firebase
  • Audio non stocké — historique en mémoire uniquement
  • Fallback texte disponible si le micro n'est pas supporté
📱 8. Installation PWA
Installer Sallon-ConnecT comme application web progressive depuis Chrome ou Edge.
  1. Démarrez Sallon-ConnecT
  2. Ouvrez http://localhost:3001 dans Chrome ou Edge
  3. Dans la barre d'adresse, cherchez l'icône d'installation
  4. Cliquez sur Installer et confirmez
La PWA affiche une page offline si le backend est indisponible. Elle ne remplace pas les scripts de démarrage.

Limites

  • Les API locales nécessitent le backend sur http://localhost:3000
  • La PWA ne stocke pas de secret
  • Un cache ancien peut afficher une version obsolète — videz le cache du site si nécessaire

💾 10. Sauvegarde et Restauration
Créer, vérifier et restaurer des sauvegardes locales de manière sécurisée.

Créer une sauvegarde

  1. Ouvrez le dashboard
  2. Allez dans Sauvegarde
  3. Lancez une création — vérifiez le nom du ZIP
powershell (API)
Invoke-RestMethod -Method POST http://localhost:3000/api/backup/create

Dry-run obligatoire avant restauration

🔒 Toujours lancer un dry-run avant toute restauration. Sans dry-run valide et sans code de confirmation, la restauration est refusée.

Ce qui n'est jamais sauvegardé

  • .env — tokens et secrets
  • node_modules/ — dépendances
  • Logs bruts et ZIP de sauvegarde existants
  • .git/, .next/, clés et certificats

Bonnes pratiques

  • Gardez les sauvegardes hors Git
  • Ne partagez jamais un ZIP sans vérifier son contenu
  • Lancez le preflight avant toute publication GitHub

Ne jamais ajouter .env, runtime, logs, backups ou dépendances.

🔒 16. Règles de Sécurité

🛡 Règles absolues — ne jamais enfreindre

  • Ne jamais publier .env dans Git ou sur GitHub
  • Ne jamais publier runtime/*.json
  • Ne jamais publier les dossiers logs/
  • Ne jamais publier les backups privés (backups/*.zip)
  • Ne jamais afficher de token, IP complète, identifiant matériel, numéro de série ou ID complet

Intégrations — opt-in uniquement

  • SmartThings est opt-in — désactivé par défaut
  • ADB est en lecture seule par défaut
  • DLNA est en découverte locale passive
  • L'assistant vocal ne lance aucune action sensible

Restauration sauvegarde

🔒 La restauration backup nécessite toujours un dry-run + confirmation explicite.

Vérification avant publication GitHub

powershell
npm run check
powershell -ExecutionPolicy Bypass -File scripts\release\preflight-github.ps1
powershell -ExecutionPolicy Bypass -File scripts\release\final-release-check.ps1
