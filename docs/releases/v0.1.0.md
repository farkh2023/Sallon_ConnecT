# Sallon-ConnecT v0.1.0

**Date de release :** 2026-05-17
**Type :** Premier prototype local complet
**Statut :** Stable — local only, no cloud required

---

## Résumé

Sallon-ConnecT v0.1.0 est le premier prototype complet d'un hub local pour piloter et observer les appareils, services multimédias, scénarios, profils et sauvegardes d'un salon connecté.

Le projet est **local-first** : aucune télémétrie, aucun cloud obligatoire, aucun secret dans Git.

Cette version consolide 25 phases de développement couvrant l'infrastructure, les intégrations locales, la qualité, l'observabilité, les profils, la sauvegarde, la documentation et l'assistant vocal.

---

## Fonctionnalités Principales

### Backend Express (Phases 1–4)
- Serveur Express sur le port 3000
- API modulaire : appareils, scénarios, scheduler, notifications, observabilité, profils, backup
- Health check local (`GET /api/health`)
- Données JSON versionnées dans `data/`

### Frontend Next.js (Phases 12–19)
- Dashboard réactif sur le port 3001
- PWA installable (mode hors-ligne partiel)
- Mode TV — interface plein écran sans clavier
- Graphes Recharts : AreaChart, RadarChart, LineChart
- Export JSON/CSV des snapshots d'observabilité
- Ancien frontend HTML statique conservé dans `public/`

### Intégrations Locales (Phases 5–11)
- **ADB** — diagnostic Android en lecture seule, aucune écriture sur l'appareil
- **DLNA/UPnP** — découverte passive uniquement, streaming assisté indexé localement
- **SmartThings** — opt-in sécurisé, clé masquée, confirmation explicite obligatoire, whitelist d'actions TV
- **Scénarios** — cinéma, travail, famille, veille, diagnostic

### Scheduler et Notifications (Phases 12–13)
- Tâches planifiées avec safelist d'actions autorisées
- Centre de notifications local avec moteur de règles
- Masquage de notifications par profil

### Observabilité Complète (Phases 19)
- Snapshots d'état système en temps réel
- Dashboard graphes temporels
- Export non-sensible JSON/CSV
- Buckets d'observabilité : memory, scheduler, notifications, security, integrations, runtime

### Profils Locaux (Phase 20)
- 5 profils : owner, family, guest, tv, diagnostic
- 17 actions contrôlées par profil
- Basculement rapide dans la TopNav
- Audit trail local des changements

## Architecture

```text
Sallon-ConnecT/
  server.js                    Backend Express — port 3000
  server/src/routes/           Modules API
  server/src/services/         Logique métier locale
  frontend/                    Application Next.js — port 3001
  data/                        Données exemple versionnées
  runtime/                     État local ignoré par Git
  backups/                     Sauvegardes locales ignorées par Git
  logs/                        Rapports locaux ignorés par Git
  scripts/windows/             Utilitaires Windows
  scripts/windows/install/     Installateur local guidé
  scripts/release/             Preflight et préparation release
  docs/                        Documentation complète
```

---

## Sécurité

- Secrets uniquement dans `.env` local (jamais dans Git)
- `.env`, `frontend/.env.local`, `runtime/*.json`, `backups/*.zip`, `logs/` et `node_modules` ignorés par `.gitignore`
- SmartThings, ADB, DLNA et streaming sont **opt-in** — désactivés par défaut
- Confirmations explicites obligatoires pour toutes les actions sensibles
- Allowlists et audits locaux pour le scheduler et SmartThings
- Les sauvegardes excluent automatiquement les secrets, dépendances, logs bruts et builds
- Les réponses API masquent les valeurs sensibles (tokens, IMEI, IPs)
- Script preflight-github.ps1 vérifie l'absence de secrets avant toute publication

## Installation Windows

### Prérequis
- Node.js 22.13 ou plus récent
- npm
- Git
- PowerShell

### Installation guidée (Phase 25)

```powershell
scripts\windows\install\install-sallon-connect.bat
scripts\windows\install\first-run-wizard.bat
```

Réparation :
```powershell
scripts\windows\install\repair-sallon-connect.bat
```

Désinstallation :
```powershell
scripts\windows\install\uninstall-sallon-connect.bat
```

### Installation manuelle

```powershell
git clone https://github.com/farkh2023/Sallon-ConnecT.git
cd Sallon-ConnecT
copy .env.example .env
npm install
cd frontend && npm install && cd ..
npm run dev
```

## Assistant Vocal Local (Phase 24)

- Reconnaissance vocale via l'API Web Speech (navigateur local uniquement)
- Commandes de navigation et de contrôle du dashboard
- Aucune donnée audio envoyée à un service externe
- Configuration dans `frontend/src/hooks/useVoiceAssistant.ts`

## Sauvegarde et Restauration (Phase 21)

- Création de ZIP locaux avec manifest SHA256
- Dry-run obligatoire avant toute restauration
- Rollback automatique créé avant restauration effective
- Code de confirmation explicite requis
- Exclusions automatiques : `.env`, `node_modules`, `.git`, clés PEM, logs bruts

```powershell
# Depuis le dashboard frontend ou l'API
POST /api/backup/create
POST /api/backup/restore   # dry-run d'abord, puis confirmation
```

## Limites Connues

- L'installateur Windows (Phase 25) ne crée pas encore de MSI natif
- ADB est en lecture seule uniquement — aucune commande d'écriture
- DLNA ne stream pas automatiquement — découverte passive uniquement
- L'assistant vocal nécessite Chrome ou Edge (API Web Speech)
- La PWA nécessite HTTPS pour l'installation complète en dehors de `localhost`
- SmartThings nécessite une clé API personnelle configurée dans `.env`
